#1. Краткое описание проекта
##Проект: Система сбора информации о компьютере (включает информацию о системе, CPU и памяти).
Назначение: Данный проект предназначен для извлечения системной информации с различных платформ (Linux и Windows). Он помогает администраторам и разработчикам получать данные о производительности и характеристиках системы.
###Технологии:
•	Java: Основной язык программирования для реализации.
•	Gradle: Инструмент для автоматизации сборки проекта.
•	JUnit и Mockito: Используются для написания и выполнения тестов.
###Пример работы:
•	На вход: Никакие входные данные не требуются, приложение извлекает информацию о системах.
•	На выход: Форматированный JSON или структура, содержащая информацию о: 
o	Модели процессора
o	Количестве ядер процессора
o	Общей памяти системы
##2. Как выполнить сборку проекта
Чтобы выполнить сборку проекта, выполните следующую команду из корневого каталога проекта:
bash
./gradlew build
Это инициирует процесс сборки, который включает компиляцию кода и тестирование. В случае успеха результаты будут сохранены в папке build.
##3. Как выполнить конфигурацию проекта
Файл конфигурации: app.properties
###Содержимое файла:
properties
memory.output.unit=MB
include.network.info=false
###Опции настройки:
•	memory.output.unit: Определяет единицу измерения для объёма памяти (MB или GB).
•	include.network.info: Указывает, включать ли сетевую информацию в сбор данных (булевое значение).
Вы можете редактировать файл app.properties, чтобы настроить параметры согласно вашим требованиям.
##4. Как запустить тесты и какой результат ожидать
Чтобы запустить тесты, выполните команду:
bash
./gradlew test
Ожидаемый результат:
•	Программа выполнит все тесты, написанные с использованием JUnit и Mockito.
•	В случае успешного прохождения тестов, на выходе будет сообщение о прохождении всех тестов. Если тесты не пройдут, Gradle предоставит подробный отчет об ошибках и упавших тестах.
##5. Как запустить приложение
Чтобы запустить приложение, выполните следующую команду:
bash
./gradlew run
Это запустит основное приложение, которое будет собирать и отображать информацию о системе в соответствии с реализованной логикой классов-коллекторов.
